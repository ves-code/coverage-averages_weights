      32             : //+PLUMEDOC INTERNAL pdbreader
      33             : /*
      34             : PLUMED can use the PDB format in several places
      35             : 
      36             : - To read molecular structure (\ref MOLINFO).
      37             : - To read reference conformations (\ref RMSD, but also many other methods in \ref dists, \ref FIT_TO_TEMPLATE, etc).
      38             : 
      39             : The implemented PDB reader expects a file formatted correctly according to the
      40             : [PDB standard](http://www.wwpdb.org/documentation/file-format-content/format33/v3.3.html).
      41             : In particular, the following columns are read from ATOM records
      42             : \verbatim
      43             : columns | content
      44             : 1-6     | record name (ATOM or HETATM)
      45             : 7-11    | serial number of the atom (starting from 1)
      46             : 13-16   | atom name
      47             : 18-20   | residue name
      48             : 22      | chain id
      49             : 23-26   | residue number
      50             : 31-38   | x coordinate
      51             : 39-46   | y coordinate
      52             : 47-54   | z coordinate
      53             : 55-60   | occupancy
      54             : 61-66   | beta factor
      55             : \endverbatim
      56             : PLUMED parser is slightly more permissive than the official PDB format
      57             : in the fact that the format of real numbers is not fixed. In other words,
      58             : any parsable real number is ok and the dot can be placed anywhere. However,
      59             : __columns are interpret strictly__. A sample PDB should look like the following
      60             : \verbatim
      61             : ATOM      2  CH3 ACE     1      12.932 -14.718  -6.016  1.00  1.00
      62             : ATOM      5  C   ACE     1      21.312  -9.928  -5.946  1.00  1.00
      63             : ATOM      9  CA  ALA     2      19.462 -11.088  -8.986  1.00  1.00
      64             : \endverbatim
      65             : 
      66             : Notice that serial numbers need not to be consecutive. In the three-line example above,
      67             : only the coordinates of three atoms are provided. This is perfectly legal and indicates PLUMED
      68             : that information about these atoms only is available. This could be both for structural
      69             : information in \ref MOLINFO, where the other atoms would have no name assigned, and for
      70             : reference structures used in \ref RMSD, where only the provided atoms would be used to compute RMSD.
      71             : 
      72             : \par Occupancy and beta factors
      73             : 
      74             : PLUMED reads also occupancy and beta factors that however are given a very special meaning.
      75             : In cases where the PDB structure is used as a reference for an alignment (that's the case
      76             : for instance in \ref RMSD and in \ref FIT_TO_TEMPLATE), the occupancy column is used
      77             : to provide the weight of each atom in the alignment. In cases where, perhaps after alignment,
      78             : the displacement between running coordinates and the provided PDB is computed, the beta factors
      79             : are used as weight for the displacement.
      80             : Since setting the weights to zero is the same as __not__ including an atom in the alignement or
      81             : displacement calculation, the two following reference files would be equivalent when used in an \ref RMSD
      82             : calculation. First file:
      83             : \verbatim
      84             : ATOM      2  CH3 ACE     1      12.932 -14.718  -6.016  1.00  1.00
      85             : ATOM      5  C   ACE     1      21.312  -9.928  -5.946  1.00  1.00
      86             : ATOM      9  CA  ALA     2      19.462 -11.088  -8.986  0.00  0.00
      87             : \endverbatim
      88             : Second file:
      89             : \verbatim
      90             : ATOM      2  CH3 ACE     1      12.932 -14.718  -6.016  1.00  1.00
      91             : ATOM      5  C   ACE     1      21.312  -9.928  -5.946  1.00  1.00
      92             : \endverbatim
      93             : However notice that many extra atoms with zero weight might slow down the calculation, so
      94             : removing lines is better than setting their weights to zero.
      95             : In addition, weights for alignment need not to be equivalent to weights for displacement.
      96             : 
      97             : \par Systems with more than 100k atoms
      98             : 
      99             : Notice that it very likely does not make any sense to compute the \ref RMSD or any other structural
     100             : deviation __using__ so many atoms. However, if the protein for which you want to compute \ref RMSD
     101             : has atoms with large serial numbers (e.g. because it is located __after__ solvent in the sorted list of atoms)
     102             : you might end up with troubles with the limitations of the PDB format. Indeed, since there are 5
     103             : columns available for atom serial number, this number cannot be larger than 99999.
     104             : In addition, providing \ref MOLINFO with names associated to atoms with a serial larger than 99999 would be impossible.
     105             : 
     106             : Since PLUMED 2.4 we allow [hybrid 36](http://cci.lbl.gov/hybrid_36/) format to be used to specify atom numbers.
     107             : This format is not particularly widespread, but has the nice feature that it provides a one-to-one mapping
     108             : between numbers up to approximately 80 millions and strings with 5 characters, plus it is backward compatible
     109             : for numbers smaller than 100000. This is not true for notations like the hex notation exported by VMD.
     110             : Using the hybrid 36 format, the ATOM records for atom ranging from 99997 to 100002 would read like these:
     111             : \verbatim
     112             : ATOM  99997  Ar      X   1      45.349  38.631  15.116  1.00  1.00
     113             : ATOM  99998  Ar      X   1      46.189  38.631  15.956  1.00  1.00
     114             : ATOM  99999  Ar      X   1      46.189  39.471  15.116  1.00  1.00
     115             : ATOM  A0000  Ar      X   1      45.349  39.471  15.956  1.00  1.00
     116             : ATOM  A0000  Ar      X   1      45.349  38.631  16.796  1.00  1.00
     117             : ATOM  A0001  Ar      X   1      46.189  38.631  17.636  1.00  1.00
     118             : \endverbatim
     119             : There are tools that can be found to translate from integers to strings and back using hybrid 36 format
     120             : (a simple python script can be found [here](https://sourceforge.net/p/cctbx/code/HEAD/tree/trunk/iotbx/pdb/hybrid_36.py)).
     121             : In addition, as of PLUMED 2.5, we provide a \ref pdbrenumber "command line tool" that can be used to renumber atoms in a PDB file.
     122             : 
     123             : */
